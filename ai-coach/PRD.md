## 一、背景与目标

### 背景

品诺筑家的销售和设计团队在日常工作中积累了大量关于客户的信息——CRM 系统里的基础数据、跟进记录、报价历史，以及大量存在于团队成员脑子里的"软信息"（客户性格、家庭情况、决策风格、顾虑点等）。

这些信息目前是分散的、非结构化的，没有被系统性地利用起来。每次谈单前，销售/设计师需要自己回忆和整理；谈单后，复盘也基本靠个人经验，缺乏系统性的分析和改进。

### 目标

构建一个 AI 辅助的客户智能系统，帮助团队：

1. **更了解客户**：把所有关于客户的信息汇聚成结构化的"客户画像"
2. **更好地谈单**：通过录音分析，找出谈单中的问题和改进点
3. **持续成长**：基于多个客户的数据，提炼出可复用的销售经验和方法论

## 二、核心模块

### 模块 1：客户画像（Customer Intelligence Profile）

**是什么**：针对每个客户，建立一份超越 CRM 基础字段的"深度档案"。

**包含内容**：
- CRM 已有数据（自动同步）：姓名、预算、户型、来源、跟进记录、报价历史
- 团队软信息（手动录入）：性格特点、家庭结构、决策风格、核心顾虑、喜好偏好
- AI 提炼的洞察（自动生成）：客户类型标签、成单概率评估、关键突破点建议

**AI 辅助录入**：
- AI 会主动提问，引导团队成员补充客户信息
- 例如："这个客户上次提到预算有限，他的决策权主要在谁手里？"
- 问题基于已有信息的空白点动态生成，不是固定问卷

**数据来源**：
```
TCB 数据库（leads/projects/quotes/followUps）
    ↓ 自动同步
客户画像基础层

团队成员手动录入 + AI 引导提问
    ↓
客户画像软信息层

谈单录音分析结果
    ↓
客户画像洞察层
```

---

### 模块 2：谈单录音分析（Conversation Analysis）

**是什么**：上传一段谈单或日常沟通的录音，AI 自动转文字并进行多维度分析。

**工作流程**：
```
上传录音文件（MP3/M4A/WAV）
    ↓
Whisper API 转文字（识别说话人）
    ↓
Claude API 分析（基于客户画像 + 对话内容）
    ↓
生成分析报告
```

**分析报告包含**：
- **对话摘要**：这次谈了什么，客户的核心关切是什么
- **客户信号**：客户表现出的兴趣点、顾虑点、情绪变化
- **销售表现评分**：倾听、需求挖掘、异议处理、成交推进各维度评分
- **问题点**：哪些地方说错了、说漏了、或者可以更好
- **改进建议**：针对这个客户，下次谈单应该怎么调整策略
- **客户画像更新**：从这次对话中提炼出的新信息，自动补充到客户画像

**关键设计**：分析是**基于客户画像的**，不是通用分析。AI 知道这个客户的背景，所以建议是针对性的，不是泛泛而谈。

---

### 模块 3：AI 教练对话（AI Coach Chat）

**是什么**：一个可以随时问的 AI 销售教练，了解你的客户，能给出针对性建议。

**使用场景**：
- 谈单前："明天要见王总，他上次说预算只有30万，我应该怎么切入？"
- 谈单后："今天谈得不顺，客户一直在说要再想想，问题出在哪？"
- 日常："这类犹豫型客户一般怎么跟进比较好？"

**技术实现**：RAG（检索增强生成）
- AI 回答时，会自动检索该客户的所有相关信息作为上下文
- 不是通用的销售建议，是基于"这个客户"的具体建议

---

### 模块 4：多客户聚合分析（Team Analytics）

**是什么**：把所有客户的数据汇总，找规律、提炼方法论。

**功能**：
- 成单客户 vs 流失客户的特征对比
- 哪类客户成单率高，哪类客户容易流失
- 团队成员的谈单能力对比分析
- 常见客户异议汇总 + 最有效的应对话术
- 自动生成"品诺筑家销售手册"（基于真实数据提炼）

**使用频率**：低频，月度/季度复盘时使用。

## 三、技术架构

### 技术选型

| 组件 | 技术方案 | 说明 |
|------|---------|------|
| 语音转文字 | OpenAI Whisper API | 支持中文，按分钟计费，约 $0.006/分钟 |
| AI 分析/对话 | Claude API（claude-sonnet-4-6） | 主力模型，深度分析用 claude-opus-4-6 |
| 向量存储 | 待定（Supabase pgvector 或 Pinecone） | 用于语义检索客户信息 |
| 文件存储 | 微信云开发 COS（已有） | 存储录音文件 |
| 数据库 | 微信云开发 TCB（已有） | 存储画像数据、分析结果 |

### 数据流

```
[录音文件] → COS 存储
     ↓
[Whisper API] → 转文字稿
     ↓
[客户画像数据] + [文字稿] → Claude API
     ↓
[分析报告] → TCB 存储 → 前端展示
     ↓
[提炼的新信息] → 更新客户画像
```

### 新增数据库集合

```
customer_profiles     客户画像（软信息层）
  - leadId            关联的线索 ID
  - softInfo          团队录入的软信息（JSON）
  - aiInsights        AI 提炼的洞察（JSON）
  - updatedAt

conversation_records  谈单录音记录
  - leadId            关联的线索 ID
  - audioUrl          录音文件 COS 地址
  - transcript        转写文字稿
  - analysis          AI 分析报告（JSON）
  - createdAt
  - createdBy

coach_sessions        AI 教练对话记录
  - leadId            关联的线索 ID（可为空，表示通用咨询）
  - messages          对话历史（JSON Array）
  - createdAt
  - createdBy
```

## 五、开发阶段规划

### Phase 1：客户画像 + 手动录入（MVP）
- 客户详情页新增"AI 分析"Tab
- 团队可以手动录入客户软信息
- AI 基于已有 CRM 数据生成初步洞察
- **不需要 Whisper，不需要向量数据库**，最简单的实现

### Phase 2：录音分析
- 支持上传录音文件
- 接入 Whisper API 转文字
- Claude 分析 + 生成报告
- 分析结果自动更新客户画像

### Phase 3：AI 教练对话
- 基于客户画像的 RAG 对话
- 接入向量数据库

### Phase 4：多客户聚合分析
- 团队级别的数据分析
- 销售手册自动生成

## 六、关键问题与风险

### 数据隐私
- 录音文件包含客户真实声音和对话内容，属于敏感数据
- 需要明确：录音只在内部使用，不对外共享
- 上传前需要告知客户（或在合同中约定）

### 成本估算（Phase 2 上线后）
- Whisper：假设每周 10 次谈单，每次 1 小时 → 约 $3.6/周 ≈ ¥26/周
- Claude API：每次分析约 $0.05-0.2 → 每周约 ¥10-50
- 总计：**每月约 ¥150-300**，可接受

### 技术复杂度
- Phase 1 复杂度低，1-2 周可完成
- Phase 2 中等，需要处理文件上传和异步任务
- Phase 3-4 复杂度高，需要向量数据库和 RAG 架构
