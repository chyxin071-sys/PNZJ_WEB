# PNZJ - 品诺筑家整装管理系统

这是一个为品诺筑家（整装公司）定制开发的轻量化、现代化的工程与客户管理系统。本项目采用“全栈+双端”的架构设计，同时包含 Web 网页端与微信小程序端，后端数据完全互通。

## 架构概览

本项目包含两个主要端点：

1. **Web 端 (`/web`)**
   - **技术栈**: Next.js 14 (App Router), React, TailwindCSS, Lucide Icons.
   - **定位**: 主要面向 PC 端后台管理，适合公司内部办公、数据总览与大屏操作。
   - **特点**: 全响应式设计（支持移动端浏览器访问），包含全局数据看板、组织架构管理、详细的客户与施工阶段管控。

2. **微信小程序端 (`/mini-program`)**
   - **技术栈**: 原生微信小程序 (WXML, WXSS, JS).
   - **定位**: 面向全员（老板、设计师、销售、项目经理）的移动端工作台。
   - **特点**: 便捷的现场打卡、客户跟进、施工节点更新与消息推送，具备细粒度的权限控制（如材料大厅仅管理员可见）。

3. **云端数据库 (WeChat CloudBase - TCB)**
   - 网页端与小程序端共享同一个微信云开发数据库。
   - Web 端通过 `tcb-js-sdk` HTTP API 路由直接访问云数据库，实现数据实时同步。

## 核心功能模块

- **客户管理 (Leads)**: 客户线索全生命周期管理（沟通中 -> 已量房 -> 方案阶段 -> 已交定金 -> 已签单）。
- **施工管理 (Projects)**: 8个标准施工节点管控，自动计算施工状态（未开工/施工中/已竣工），健康度预警（正常/预警/严重延期）自动置顶。
- **报价管理 (Quotes)**: 灵活的报价单生成工具，直接关联客户与材料大厅，支持随时查看与编辑。
- **材料大厅 (Materials)**: 企业标准化主辅材、软装及套餐库，支持出入库记录与库存动态管理。
- **团队协同待办 (Todos)**: 跨角色的任务派发与跟进，支持按人员、状态、日期多维度筛选。
- **全局看板 (Dashboard)**: 宏观的数据统计面板，提供转化率、施工进度及业绩总览。

## 部署说明

### Web 端部署 (微信云托管)

本项目已配置好适用于**微信云托管 (WeChat Cloud Run)** 的环境。

1. 在 `web/` 目录下包含 `Dockerfile` 和 `.dockerignore`。
2. 可以在微信开发者工具的云托管控制台中，授权关联本 GitHub 仓库。
3. 选择 `web/` 作为构建目录，云托管会自动监听 `main` 分支的提交并触发流水线部署。

### 本地开发运行

**运行 Web 端**:
```bash
cd web
npm install
npm run dev
# 访问 http://localhost:3000
```

**运行 小程序端**:
使用 [微信开发者工具] 导入 `mini-program` 目录，填入您的 AppID，并确保已开通云开发环境。

## 权限控制说明

系统内置了静态的角色权限系统（Role-Based Access Control）：
- `admin` (管理员): 拥有所有模块的最高权限，可查看组织架构，进行全局材料库管理和数据删除。修改员工信息、跨角色派发待办任务。
- `sales` / `designer` (销售/设计): 可管理自己名下的客户线索与报价，可见全局数据但涉密信息（如客户电话）会对非负责人脱敏。
- `manager` (项目经理): 专注工地管理，可更新施工节点、申请材料及填报售后记录。

*注意：小程序端直接调用云数据库存在权限校验，修改其他员工信息或敏感信息时，需在云开发控制台中将对应集合（如 `users`）设为“自定义权限”，并将 `write` 设置为 `true`。*
