## 三、共享组件设计

### 3.1 FolderSelectModal 组件

**位置：** `web/src/components/FolderSelectModal.tsx`

**功能：**
- 显示文件夹列表（单选）
- 可见性选择（内部/公开）
- 支持新建文件夹
- 确认后返回选择结果

**接口设计：**
```typescript
interface FolderSelectModalProps {
  isOpen: boolean;
  onClose: () => void;
  onConfirm: (folder: string, visibility: 'internal' | 'public') => void;
  folders: string[];  // 现有文件夹列表
  defaultFolder?: string;  // 默认选中的文件夹
  defaultVisibility?: 'internal' | 'public';  // 默认可见性
  leadId: string;  // 用于更新数据库中的 fileFolders
}
```

**交互流程：**
1. 用户选择文件后，弹出此模态框
2. 用户选择文件夹（或输入新文件夹名称）
3. 用户选择可见性（默认"内部"）
4. 点击确认，回调返回 `(folder, visibility)`
5. 如果是新文件夹，组件内部会先更新 `leads.fileFolders`，再回调

**UI 布局：**
```
┌─────────────────────────────┐
│  选择文件夹                    │
├─────────────────────────────┤
│  ○ 默认文件夹                 │
│  ○ 平面布局                   │
│  ○ 效果图                     │
│  ○ 新建文件夹...              │
│    └─ [输入框: 文件夹名称]    │
├─────────────────────────────┤
│  可见性：                     │
│  ● 内部  ○ 公开               │
├─────────────────────────────┤
│        [取消]  [确认]         │
└─────────────────────────────┘
```

### 4.1 功能 1：项目资料文件夹（编号 36）

### 4.3 功能 3：施工节点子工序排期（编号 34）

**文件：** `web/src/app/projects/[id]/page.tsx`

**本阶段改动内容：**

#### 4.3.1 编辑模式增强
- 编辑大节点时，可以为每个子工序设置：
  - 预计开工日期（日期选择器）
  - 工期天数（数字输入）
  - 自动计算预计完工日期（开工日期 + 工期天数）

#### 4.3.2 子工序详情浮窗
- 点击子工序名称，弹出浮窗
- 显示：

- 提供"开始"和"验收完成"按钮

#### 4.3.3 实际时间记录
- 子工序开始时记录 `actualStartDate = 当前日期`
- 子工序验收时记录 `actualEndDate = 当前日期`
- 自动计算已用天数

### 4.4 功能 4：查看申请管理（编号 39）

**文件：** `web/src/app/projects/[id]/page.tsx`

**改动内容：**

#### 4.4.1 新增 tab
- 在"基本信息"、"施工进度"后面加"查看申请"tab
- tab 标签显示未读数量（如"查看申请 (3)"）
- 有未读申请时显示红点

- 从 `shareAccess` 集合查询 `projectId === 当前工地ID` 的申请记录

- 显示：申请人姓名、手机号（脱敏）、申请时间、状态

- 点击申请记录，弹出详情模态框

- 提供"通过"和"拒绝"按钮
- 调用 API 更新 `shareAccess` 记录的 `status` 字段
- 发送通知给申请人（复用现有通知系统）

#### 4.4.4 红点逻辑
- 有 `status === 'pending'` 的申请时显示红点和数量
- 进入 tab 后自动标记已读（可选：在 `shareAccess` 加 `staffReadAt` 字段）

## 五、API 设计

### 5.1 文件上传 API（修改现有）
**路径：** `POST /api/upload`
**改动：** 接收 `folder` 和 `visibility` 参数，写入文件记录

### 5.2 文件列表 API（修改现有）
**路径：** `GET /api/leads/[id]/files`
**改动：** 返回的文件记录包含 `folder` 和 `visibility` 字段

### 5.3 更新客户文件夹列表
**路径：** `PUT /api/leads/[id]`
**改动：** 支持更新 `fileFolders` 字段

### 5.4 查看申请列表（新增）
**路径：** `GET /api/projects/[id]/share-access`
**功能：** 查询该工地的所有申请记录
**返回：** `shareAccess` 集合中 `projectId` 匹配的记录数组

### 5.5 审批申请（新增）
**路径：** `PUT /api/share-access/[id]`
**功能：** 更新申请状态（approved/rejected），发送通知给申请人
**参数：** `{ status: 'approved' | 'rejected', reviewedBy: string }`
