# 轻量化整装全链路管理系统 (V1.0) PRD

## 1. Executive Summary (执行摘要)

我们正在为20人规模的小微型整装公司构建一个**“线索-签约-施工-收款”闭环的轻量化管理系统**。解决市面通用CRM过于冗余复杂、且销售与施工数据断层的问题。本系统旨在提升销售与设计师的协作效率，实现施工节点透明化，并通过极简的数据看板辅助老板进行业务决策。

## 2. Problem Statement (问题定义)

- **痛点1：市面系统过于臃肿**：通用CRM包含太多无关行业（如复杂的审批流、多级权限），增加学习成本，导致员工抗拒使用。
- **痛点2：数据断层**：前端销售谈单与后端施工进度脱节。施工经理不知道前期承诺，销售不知道工地进度，老板无法全局把控。
- **痛点3：数据隐私与内耗**：小团队容易发生“飞单”或抢单，缺乏合理的权限隔离与分配机制。

## 3. Target Users & Personas (目标用户与角色)

- **管理员 (老板/店长)**：全局视角。关注签单率、漏斗转化、核心财务数据（营收、回款）、施工延期率。拥有所有权限（包括跨级分配线索、指派工长等）。
- **销售员 (客户经理)**：录入线索，分配设计师，跟进客户意向。只能看到自己及参与跟单的客户。
- **设计师**：被分配线索后介入。记录设计沟通需求，上传图纸/方案，协助促成签单。与销售共享对应客户的数据。
- **施工经理 (工长)**：负责已签单落地的施工项目。在核心节点进行打卡、上传现场照片。

## 4. 业务流程逻辑 (Workflow)

1. **线索期**：销售录入线索 → 持续跟进。
2. **派单期**：系统支持**销售自行分配**或**老板统一派单**给指定设计师 → 设计师介入，共同跟进。
3. **方案与报价期**：设计师/销售输出【报价单】与【主材选择】，与客户确认细节。
4. **转化期**：确认签单 → 必须填写【合同总价】与【收款计划】 → 状态变更为“已签单”。
5. **交付期**：系统自动生成施工项目 → **老板/店长指派**施工经理 → 施工经理按详细节点打卡传图。
6. **收款与归档**：各节点触发对应阶段的收款确认（首付、中期、尾款） → 竣工验收 → 归档。

## 5. 核心功能模块 (Solution Overview)

### 5.1 登录与权限模块 (基石)

- 基于 UserID 和 Role 的严格数据隔离机制。
- 不同角色登录后渲染不同的侧边栏与 Dashboard。

### 5.2 客户/线索模块 (CRM)

- **客户池**：支持录入、分配设计师（销售与老板均有分配权限）、跟进记录（区分销售跟进和设计沟通）。
- **状态流转**：线索 → 意向 → 方案/报价 → 签单/流失。

### 5.3 报价与材料模块 (核心新增)
- **报价单管理**：在客户意向阶段，支持录入或上传结构化的报价单（包含总价、明细等），作为签单的依据。
- **主材管理**：记录客户的“主材选择”确认单，并独立管理“材料进场”确认（不与现场施工打卡节点强绑定），防止施工延误。

### 5.4 施工项目模块 (ERP)
- **自动流转**：客户变更为“已签单”后自动生成施工项目。工长由老板/店长指派。
- **精细化节点管理**：采用符合整装实际的 11 个顺序节点：
  `拆除` → `水电交底` → `水电` → `木工` → `瓦工` → `美缝` → `界面剂` → `墙面` → `门（金属门、木门）` → `定制` → `大保洁`。
- **节点打卡**：必须上传照片并确认完成，才能解锁下一节点。

### 5.5 极简财务模块 (前置占位)

- 合同总价 (contract_amount)、收款计划 (collection_plan)、已收金额 (collected_amount)。
- 手动确认节点回款（如：水电完工后，一键确认收到中期款）。

### 5.6 全局看板 (Dashboard)

- **宏观数据**：本月新增线索、签单数、在建工地数。
- **光荣榜**：销售/设计师签单排名（脱敏展示）。
- **跑马灯播报**：实时播报签单或验收喜报（隐藏客户全名，如“恭喜设计师李某某与张先生成功签约”）。

## 6. UI/UX 设计要求与技术栈

- **核心技术栈**：React/Next.js + TailwindCSS (前端)，Node.js/Supabase/PostgreSQL (后端与数据库)。
- **平台适配**：全功能响应式设计（Mobile First）。确保老板随时看数据、销售在外跟进、工长在现场传图都拥有极致的移动端体验。
- **视觉风格**：Magazine Style（杂志级排版），极简高质感。
- **色彩与字体**：黑白灰主基调，大面积留白；仅在预警或待办处使用莫兰迪色系点缀；无衬线字体。

## 7. Out of Scope (暂不开发范围)

- 复杂的财务发票与对公流水管理（保持轻量，仅记录数字）。
- 客户/业主端小程序（V1.0优先解决内部协作，后续可考虑增加业主展示页）。
- 复杂的自定义审批流（采用固定流程驱动，避免审批卡点）。
