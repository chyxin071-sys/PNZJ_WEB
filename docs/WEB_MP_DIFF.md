## 三十四、工地详情 - 施工节点编辑功能与小程序差异大（高优先级）🔴 待修复

### 小程序端（已实现）
`projectDetail/index.js` 的节点编辑功能：
- 编辑模式下可修改大节点名称、子工序名称、子工序工期（天数）
- 支持拖拽排序大节点
- 支持新增/删除大节点和子工序
- 编辑排期弹窗：填写预计开工日期 + 工期天数，自动计算预计完工日期
- 子工序浮窗：显示计划/实际开始、完工时间，用时天数
- 验收弹窗：可上传现场照片 + 填写备注
- 延期弹窗：填写延期天数和原因

### 网页端（未实现/差异大）
`projects/[id]/page.tsx` 的节点编辑功能不完整，缺少：
- 子工序级别的排期编辑（预计开工 + 工期天数）
- 实际开工/完工时间记录
- 验收时上传现场照片
- 延期记录功能
- 节点拖拽排序

**影响**：工地管理核心功能在网页端严重缺失，项目经理无法在网页端完整操作施工进度。

## 三十八、消息通知 - 已读状态不跨端实时同步（设计限制）⚠️ 已知限制

### 说明
两端的已读状态都存储在数据库 `notifications.isRead` 字段，理论上是同步的。但网页端只有每 60 秒轮询一次（`MainLayout.tsx:72`），不支持实时推送。因此：
- 小程序标记已读后，网页端需等待最多 60 秒才会刷新
- 这是架构限制（无 WebSocket），不是 bug，可接受
