## 写作计划

1. 先写完整的大纲（已完成）
2. 逐个文档填充内容
3. 每个文档独立完成，不要跳跃
4. 确保每个文档都足够详细
5. 从真实项目中提取通用经验

## 注意事项

- 不涉及具体项目的业务细节
- 用通俗易懂的语言
- 配合实例和代码
- 标注常见的坑
- 提供可行的解决方案
