# AI 辅助开发路线图与流程指南 (Development Plan)

基于“一人即团队”的 AI 辅助开发模式，我们采用**“视觉驱动 + 模块化增量”**的敏捷开发流程。避免一次性丢给 AI 太多需求导致其逻辑混乱。

## 第一阶段：需求对齐与结构化 (已完成)

- **目标**：明确产品边界，统一数据流转与角色权限逻辑。
- **输出**：`PRD.md`（当前目录下的产品需求文档）。
- **确认点**：数据库建模前，确保所有实体关系（如 User, Lead, Project）不再变更。

## 第二阶段：UI 雏形与交互预览 (视觉与交互驱动) - [已基本完成]

不要直接写后端！先让 AI 产出前端页面，使用假数据（JSON）填充，验证“杂志级” UI 和基础交互逻辑。

- **目标**：确定视觉风格（Magazine Style）、菜单布局、核心页面操作体验。
- **已完成任务**：
  1. **全局看板 (Dashboard)**：包含带换行自定义标签的饼图、联动全局年份/月份的柱状图折线图、移动端自适应。
  2. **核心业务模块UI**：线索管理 (A/B/C/D评级)、报价管理 (灵活手写+库管)、合同管理、施工管理 (ERP)。
  3. **交互细节打磨**：
     - 各页面统一的悬浮高级筛选框。
     - 统一风格的危险操作（删除）按钮与气泡二次确认（置于页面右上方）。
     - 项目详情页在移动端的 Flex `order` 极致重排与文件默认折叠体验。
     - 全局 CustomerDocuments 客户文件共享组件。
  4. **Mock 数据机制**：列表数据均通过本地结构读取、聚合。

## 第三阶段：数据库设计与权限基石 (后端接入大厂云开发 Serverless) - [进行中]

在 UI 定稿后，开始让 AI 构建后端与数据库。这是整个系统的骨架。
**技术选型更新**：为了保证国内网络访问的绝对稳定、数据合规，以及未来可能的小程序拓展，我们决定采用**大厂（腾讯云云开发 CloudBase 或 阿里云 Serverless）**方案，彻底免除服务器运维烦恼。

- **目标**：打通云端数据库，实现真实的用户注册、登录，建立严密的 Role-based 权限控制。
- **任务拆解**：
  1. **云环境初始化**：
     - 在腾讯云/阿里云开通云环境，获取 `Env ID` 等核心密钥。
  2. **数据库建模 (NoSQL / SQL 转换)**：
     - 根据 `schema.sql`，在云开发的控制台创建真实的数据表/集合（User集合, Lead集合, Project集合 等）。
     - 重点确认：权限控制字段（Role）以及行级安全策略 (Security Rules)。
  3. **核心接口联调**：
     - `云函数：登录鉴权` (手机号/密码或未来微信一键登录)
     - `GET /api/user/me` (获取当前用户身份与角色，以此渲染不同的侧边栏菜单)
  4. **铁律测试**：不登录绝对无法访问内部页面；销售 A 无法通过接口获取销售 B 的线索。

## 第四阶段：功能模块“逐个击破” (前后端联调)

按照业务流的自然顺序，逐一替换第二阶段的 JSON 假数据为真实的后端 API 请求。

- **目标**：打通数据闭环。
- **任务拆解**：
  1. **CRM 模块 (线索/客户)**：
     - 接口：线索增删改查。
     - 逻辑：分配设计师接口（更新 Lead 表的 `designer_id`）。
  2. **流转引擎 (签单动作)**：
     - 接口：点击“签单”时，保存【合同金额】与【收款计划】，并**自动在 Project 表生成一条施工记录**。
  3. **ERP 模块 (施工打卡)**：
     - 接口：节点更新、图片上传（可先传到服务器本地或使用第三方 OSS）。
  4. **看板聚合 (Dashboard)**：
     - 接口：编写 SQL 聚合查询（统计本月签单额、跑马灯最新动态等）。

## 第五阶段：部署与内部测试

- **目标**：让 20 人的团队用起来。
- **任务拆解**：
  1. 部署到云服务器（如 阿里云/腾讯云 的轻量应用服务器）。
  2. 导入公司第一批真实的员工账号（User表初始化）。
  3. 进行第一周的试运行，收集反馈进行微调。
