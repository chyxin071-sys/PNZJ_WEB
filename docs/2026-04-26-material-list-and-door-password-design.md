# 主材清单优化与入户密码功能设计文档

**日期：** 2026-04-26
**版本：** v1.0
**状态：** 已批准

## 概述

本文档描述两个功能的设计方案：

1. 主材清单功能优化 - 重组表单字段、优化卡片展示、增加可见性控制
2. 入户密码功能 - 在客户资料中新增入户密码字段，方便施工人员查看

## 一、主材清单功能优化

### 1.1 背景

现有主材清单功能已实现基础架构，但存在以下问题：

- 表单字段不够细化，不同材料类型需要不同的字段组合
- 卡片展示信息层次不清晰，难以快速定位关键信息
- 缺少客户端可见性控制

### 1.2 设计目标

- 按7个材料分类定制表单字段，提升录入效率
- 优化卡片展示，按材料类型突出核心信息
- 增加整体可见性开关，控制客户端是否可见
- 为"其他"分类添加智能提示，减少录入错误

### 1.4 卡片展示设计

### 1.5 权限和可见性控制

#### 1.5.1 员工端（小程序）

- **查看权限**：所有员工可查看主材清单
- **编辑权限**：相关人员（销售/设计师/签单人/项目经理）+ 管理员可以：
  - 添加/编辑/删除材料记录
  - 切换整个清单的"公开/仅内部"状态
- **UI设计**：
  - 主材清单 tab 顶部右侧显示可见性开关
  - 开关样式：切换按钮，显示当前状态（"公开" / "仅内部"）
  - 只有有编辑权限的人能看到并操作开关

#### 1.5.2 客户端（分享页）

- **公开状态**：显示所有材料记录（只读模式）
- **仅内部状态**：显示空状态"暂无主材记录"（不提示"未公开"等字样）

#### 1.5.3 数据存储

- 在 `leads` 集合中新增字段：`materialListVisible`（布尔值，默认 `false`）
  - `true` = 公开（客户可见）
  - `false` = 仅内部（客户不可见）

### 1.6 验证规则

采用宽松验证策略，方便快速录入：

- **必填字段**：
  - 所有分类：区域（必填）
  - 集成吊顶/电器：名称（必填）
  - 其他：材料（必填）
- **可选字段**：品牌、型号、规格、数量、备注等均为可选
- **验证时机**：点击"保存"按钮时验证必填字段

### 1.7 技术实现要点

#### 1.7.1 表单动态渲染

- 根据 `materialForm.category` 动态显示对应字段
- 使用 `wx:if` / `wx:elif` 条件渲染不同分类的表单

#### 1.7.2 智能提示实现

- 监听"其他"分类的材料选择事件
- 根据选择的材料名称，动态显示对应提示文字
- 提示文字显示在备注输入框上方，使用浅色背景和图标

#### 1.7.3 卡片渲染优化

- 在 `computeGroupedMaterials()` 中为每条记录添加 `displayInfo` 字段
- 根据 `category` 预处理卡片展示内容，避免 WXML 中复杂逻辑

#### 1.7.4 可见性开关

- 在 `projectFiles/index.wxml` 主材清单 tab 顶部添加开关组件
- 调用 API 更新 `leads.materialListVisible` 字段
- 分享页根据该字段决定是否显示材料清单

## 二、入户密码功能

### 2.1 背景

施工人员需要入户密码才能进入业主家中施工，该信息通常在签单后、开工前获取。目前系统中没有专门字段记录，导致信息分散在备注或聊天记录中，查找不便。

### 2.2 设计目标

- 在客户资料中新增入户密码字段
- 在客户详情页和工地详情页显示入户密码
- 只有相关人员和管理员可见可编辑

### 2.3 数据存储

- **字段名**：`doorPassword`
- **存储位置**：`leads` 集合
- **字段类型**：字符串（可选）
- **默认值**：空字符串或 `null`

### 2.4 显示位置

#### 2.4.1 客户详情页（小程序 leadDetail）

- **位置**：资料卡片中，小区地址字段下方单独一行
- **格式**：`入户密码：123456`
- **显示方式**：明文显示（不需要隐藏/显示切换）
- **权限**：只有相关人员和管理员可见

#### 2.4.2 工地详情页（小程序 projectDetail）

- **位置**：资料卡片中，地址字段下方单独一行
- **格式**：`入户密码：123456`
- **显示方式**：明文显示
- **权限**：只有相关人员和管理员可见

### 2.5 编辑方式

#### 2.5.1 客户详情页

- 点击底部"编辑资料"按钮
- 进入编辑模式，表单中新增"入户密码"字段
- 与其他字段（姓名、电话、地址等）一起保存

#### 2.5.2 表单设计

- **字段标签**：入户密码
- **输入类型**：文本输入框
- **占位符**：请输入入户密码
- **验证规则**：可选字段，无格式限制

### 2.6 权限控制

#### 2.6.1 可见可编辑

- 客户的销售人员
- 客户的设计师
- 客户的签单人
- 工地的项目经理
- 系统管理员（role === 'admin'）

#### 2.6.2 不可见

- 其他无关员工

#### 2.6.3 权限判断逻辑

```javascript
const userInfo = wx.getStorageSync('userInfo');
const myName = userInfo ? userInfo.name : '';
const isAdmin = userInfo && userInfo.role === 'admin';

// 客户详情页
const isLeadRelated = lead.sales === myName || lead.designer === myName || lead.signer === myName;
const canSeeDoorPassword = isAdmin || isLeadRelated;

// 工地详情页
const isProjectRelated = project.manager === myName;
const canSeeDoorPassword = isAdmin || isProjectRelated || isLeadRelated;
```

### 2.7 技术实现要点

#### 2.7.1 数据库更新

- 在 `leads` 集合中新增 `doorPassword` 字段（可选）
- 无需数据迁移，新字段默认为空

#### 2.7.2 客户详情页修改

- `mini-program/pages/leadDetail/index.wxml`：资料卡片中新增入户密码显示
- `mini-program/pages/leadDetail/index.js`：编辑表单中新增 `doorPassword` 字段

#### 2.7.3 工地详情页修改

- `mini-program/pages/projectDetail/index.wxml`：资料卡片中新增入户密码显示
- `mini-program/pages/projectDetail/index.js`：从关联的 `lead` 中读取 `doorPassword`

#### 2.7.4 API 修改

- `/api/leads/[id]/route.ts`：PUT 请求支持更新 `doorPassword` 字段

## 五、主材清单图片上传与多步确认机制（2026-04-26 需求更新）

### 5.1 需求背景与目标
- **图片上传（带压缩）**：主材卡片需支持图片上传功能，上传时需进行压缩。以1行4列的缩略图展示，并带有单独的编辑/确认状态。
- **全局多选发送**：不再是按单项发送。在主材清单最底部放置全局“发送给客户确认”按钮。点击后可多选分类（大项）一次性发送。
- **客户待确认与确认状态**：
  - 员工发送后，该分类进入“待确认”状态。
  - 客户分享页收到后，底部显示绿色的“待确认”按钮。
  - 客户确认后，状态变为“已确认”，并在主材清单卡片下方展示“客户已确认 | 确认时间”。
  - 允许员工重复发送以覆盖旧版本，重复发送后状态重新变为“待确认”。
- **跟进记录与多端通知**：
  - 客户确认后，自动添加系统跟进记录。
  - 员工发送给客户时，客户收到微信订阅消息。
  - 客户点击确认后，员工收到微信订阅消息及小程序内部通知。
  - 完善权限审批流：客户申请查看工地通过时，客户需收到微信订阅消息（因此申请时需触发授权弹窗）。

### 5.2 数据结构变更设计
- `leads` 集合中的 `materialList` 数组内的每个对象新增字段：
  - `images: string[]`（图片云存储URL数组，默认为空数组）
- `leads` 集合新增顶层字段：
  - `clientMaterialSnapshots: Object`（存储发给客户的快照）
    - 键名为大项分类名称（如 `"瓷砖/木地板"`）
    - 键值为：`{ items: Array, status: 'pending' | 'confirmed', sentAt: Number, confirmedAt: Number }`

### 5.3 UI 与交互设计

#### 5.3.1 员工端 - 卡片图片管理
- **展示区**：在卡片“备注”下方，展示 `images` 缩略图（`grid-cols-4`）。点击可全屏预览。
- **状态切换**：默认只读。点击编辑进入编辑模式，可删除和追加上传（调用 `wx.chooseMedia` 并开启压缩）。编辑完点击确认保存。

#### 5.3.2 员工端 - 全局发送客户确认
- **入口**：主材清单 Tab 最底部悬浮固定按钮“发送给客户确认”。
- **发送交互**：点击后弹出底部 ActionSheet（多选列表），列出所有包含材料的分类。
  - 若该分类已发送过，列表项中会显示“（已确认/待确认）”。
  - 勾选要发送的分类后，点击确定。系统深拷贝选中的分类数据写入 `clientMaterialSnapshots`，状态设为 `pending`，并更新 `sentAt`。
  - 触发发送微信订阅消息给该客户绑定的微信号。
- **列表页状态展示**：在员工端的主材清单列表中，已发送的分类标题下方展示如：“客户已确认 | 确认时间: MM-DD HH:mm”或“待客户确认 | 发送时间: MM-DD HH:mm”。

#### 5.3.3 客户端 - 确认交互
- **展示**：客户进入分享页的材料清单Tab，渲染 `clientMaterialSnapshots` 中的数据。
- **操作**：如果存在 `status === 'pending'` 的分类，页面最底部悬浮显示绿色“待确认”按钮。
- **确认逻辑**：客户点击“待确认”，弹出确认弹窗。确认后：
  1. 对应分类的状态变更为 `confirmed`，更新 `confirmedAt`。
  2. 后台自动为该客户添加一条跟进记录（内容：“客户已确认主材清单：[分类名称]”）。
  3. 触发微信订阅消息与小程序内部通知给员工（销售/设计师/项目经理等）。

#### 5.3.4 废除全局可见性
- 移除顶部“客户端可见性（公开/仅内部）”开关及相关逻辑。

### 5.4 微信订阅消息补充（模板复用 PROJECT_UPDATE）
- 客户申请工地访问时：调起 `requestSubscribeMessage` 授权。审批通过时发送通知给客户。
- 员工发送材料清单时：客户收到待确认通知。
- 客户确认材料清单时：员工收到确认通知。
