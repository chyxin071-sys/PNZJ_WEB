## 2. 小程序端部署

### 2.1 上传代码

**步骤**：
1. 打开微信开发者工具
2. 确认代码无错误
3. 点击右上角"上传"按钮
4. 填写版本号和版本说明
   ```
   版本号：v1.1.0
   版本说明：
   - 新增客户查看权限申请功能
   - 修复跟进记录排序问题
   - 优化订阅消息授权流程
   ```
5. 点击"上传"

### 2.2 提交审核

**步骤**：
1. 登录小程序后台
2. 版本管理 → 开发版本
3. 找到刚上传的版本
4. 点击"提交审核"
5. 填写审核信息：
   - 配置功能页面（选择主要功能页面）
   - 测试账号（提供测试账号和密码）
   - 补充说明（说明主要功能和使用场景）
6. 提交审核

**审核时间**：
- 通常 1-3 个工作日
- 节假日可能延长

**审核注意事项**：
- 确保所有功能正常
- 测试账号可以正常登录
- 不要有违规内容
- 不要有未完成的功能

### 2.3 发布上线

**步骤**：
1. 审核通过后会收到通知
2. 登录小程序后台
3. 版本管理 → 审核版本
4. 点击"发布"
5. 确认发布

**发布后**：
- 用户会在 24 小时内收到更新
- 可以在"线上版本"中看到当前版本
- 可以回退到上一个版本（如果有问题）

### 2.4 版本回退

**场景**：发布后发现严重 Bug

**步骤**：
1. 版本管理 → 线上版本
2. 点击"版本回退"
3. 选择要回退到的版本
4. 确认回退

**注意**：
- 只能回退到上一个版本
- 回退后需要重新修复 Bug 并提审

## 3. Web端部署

### 3.1 使用 Vercel 部署（推荐）

**优点**：
- 免费（个人项目）
- 自动部署（Git 推送后自动构建）
- 自动 HTTPS
- CDN 加速
- 零配置

**步骤**：

**1. 安装 Vercel CLI**
```bash
npm i -g vercel
```

**2. 登录 Vercel**
```bash
vercel login
```

**3. 部署**
```bash
cd web
vercel
```

**4. 配置环境变量**
- 在 Vercel 控制台 → Settings → Environment Variables
- 添加以下变量：
  ```
  TCB_ENV_ID=你的环境ID
  TCB_SECRET_ID=你的SecretID
  TCB_SECRET_KEY=你的SecretKey
  JWT_SECRET=你的JWT密钥
  ```

**5. 重新部署**
```bash
vercel --prod
```

**6. 绑定域名（可选）**
- Vercel 控制台 → Settings → Domains
- 添加自定义域名
- 配置 DNS 解析

### 3.2 使用自己的服务器部署

**环境要求**：
- Node.js 18+
- PM2（进程管理）
- Nginx（反向代理）

**步骤**：

**1. 安装依赖**
```bash
cd web
npm install
```

**2. 构建**
```bash
npm run build
```

**3. 配置环境变量**
```bash
# 创建 .env.production 文件
TCB_ENV_ID=你的环境ID
TCB_SECRET_ID=你的SecretID
TCB_SECRET_KEY=你的SecretKey
JWT_SECRET=你的JWT密钥
```

**4. 启动服务**
```bash
# 使用 PM2 管理进程
npm i -g pm2
pm2 start npm --name "cm1-web" -- start
pm2 save
pm2 startup
```

**5. 配置 Nginx**
```nginx
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**6. 配置 HTTPS（使用 Let's Encrypt）**
```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d 你的域名

# 自动续期
sudo certbot renew --dry-run
```

## 4. 云函数部署

### 4.1 部署步骤

**方法1：在开发者工具中部署**
1. 打开微信开发者工具
2. 找到云函数目录（cloudfunctions）
3. 右键云函数文件夹
4. 选择"上传并部署：云端安装依赖"
5. 等待部署完成

**方法2：在云开发控制台部署**
1. 登录云开发控制台
2. 云函数 → 函数列表
3. 点击"新建云函数"
4. 上传代码包（zip）
5. 配置运行环境（Node.js 16）
6. 保存并部署

### 4.2 需要部署的云函数

**sendSubscribeMessage**：
- 功能：发送订阅消息
- 依赖：无
- 环境变量：APPID, APPSECRET

**getPhoneNumber**：
- 功能：获取用户手机号
- 依赖：无
- 环境变量：APPID, APPSECRET

**login**：
- 功能：获取用户 openid
- 依赖：无
- 环境变量：APPID, APPSECRET

### 4.3 云函数配置

**环境变量配置**：
1. 云开发控制台 → 云函数 → 选择函数
2. 函数配置 → 环境变量
3. 添加以下变量：
   ```
   APPID=你的小程序AppID
   APPSECRET=你的小程序AppSecret
   ```

**超时时间配置**：
- 默认 3 秒
- 如果函数执行时间较长，可以调整到 20 秒

**内存配置**：
- 默认 256MB
- 通常够用，如果不够可以调整到 512MB

## 5. 数据库配置

### 5.1 创建集合

**在云开发控制台创建以下集合**：
```
users, leads, projects, quotes, contracts,
materials, inventory, followUps, todos,
notifications, shareAccess
```

### 5.2 配置权限

**所有集合设置为"所有用户可读写"**：
1. 云开发控制台 → 数据库
2. 选择集合 → 权限设置
3. 选择"所有用户可读写"
4. 保存

**注意**：
- 这样设置安全性较低
- 生产环境建议用云函数 + 权限校验
- 或者设置为"仅创建者可读写" + 云函数操作

### 5.3 创建索引（可选）

**提升查询性能**：
```javascript
// 在云开发控制台 → 数据库 → 索引管理

// users 集合
{ "account": 1 }  // 唯一索引

// leads 集合
{ "customerNo": 1 }  // 唯一索引
{ "status": 1 }
{ "manager._id": 1 }
{ "lastFollowUpAt": -1 }

// projects 集合
{ "projectNo": 1 }  // 唯一索引
{ "leadId": 1 }
{ "status": 1 }
```

### 5.4 初始化数据

**创建管理员账号**：
```json
{
  "account": "admin",

  "name": "管理员",
  "role": "admin",
  "phone": "13800138000",
  "status": "active",
  "createdAt": { "$date": 1714147200000 }
```

## 6. 域名与SSL配置

### 6.1 域名配置

**小程序端**：
1. 小程序后台 → 开发 → 开发设置
2. 服务器域名 → request 合法域名
3. 添加以下域名：
   ```
   https://你的Web端域名
   https://tcb-api.tencentcloudapi.com
   ```

**Web端**：
- 如果使用 Vercel，自动配置
- 如果使用自己的服务器，需要配置 DNS 解析

### 6.2 SSL证书配置

**使用 Let's Encrypt（免费）**：
```bash
sudo certbot --nginx -d 你的域名
```

**使用付费证书**：
1. 购买 SSL 证书
2. 下载证书文件
3. 配置到 Nginx

## 9. 监控与日志

### 9.1 小程序端监控

**微信小程序后台**：
- 统计 → 实时统计（访问量、用户数）
- 运维中心 → 性能监控（启动耗时、网络请求）
- 运维中心 → 错误日志（JS 错误、API 错误）

**云开发控制台**：
- 云函数 → 日志（查看云函数调用日志）
- 数据库 → 监控（查看数据库读写次数）
- 存储 → 监控（查看存储使用量）

### 9.2 Web端监控

**Vercel 控制台**：
- Analytics（访问量、性能）
- Logs（部署日志、运行日志）

**自己的服务器**：
```bash
# PM2 日志
pm2 logs cm1-web

# Nginx 日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log
```

### 9.3 告警配置

**云开发告警**：
- 云开发控制台 → 监控告警
- 配置告警规则（如云函数错误率 > 10%）
- 配置告警接收人（邮箱、手机）

## 10. 备份与恢复

### 10.1 数据库备份

**手动备份**：
1. 云开发控制台 → 数据库
2. 选择集合 → 导出
3. 选择导出格式（JSON/CSV）
4. 下载备份文件

**自动备份**：
- 云开发自动备份（保留 7 天）
- 可以在控制台恢复

### 10.2 代码备份

**Git 仓库**：
- 代码已托管在 Git
- 定期推送到远程仓库

**云存储备份**：
- 定期下载云存储文件
- 保存到本地或其他云存储

### 10.3 数据恢复

**从备份恢复**：
1. 云开发控制台 → 数据库
2. 选择集合 → 导入
3. 选择备份文件
4. 确认导入

**从自动备份恢复**：
1. 云开发控制台 → 数据库 → 备份
2. 选择备份时间点
3. 恢复到新集合或覆盖原集合

## 11. 故障处理

### 11.1 常见故障

**小程序无法打开**：
- 检查服务器域名配置
- 检查云开发环境ID
- 查看错误日志

**Web端无法访问**：
- 检查服务器是否运行
- 检查域名解析
- 检查 Nginx 配置
- 查看服务器日志

**云函数调用失败**：
- 检查云函数是否部署
- 检查环境变量配置
- 查看云函数日志

**数据库操作失败**：
- 检查数据库权限
- 检查网络连接
- 查看错误日志

### 11.2 故障排查流程

```
1. 确认故障现象
   ↓
2. 查看错误日志
   ↓
3. 定位问题原因
   ↓
4. 修复问题
   ↓
5. 验证修复结果
   ↓
6. 记录故障和解决方案
```
