## 📝 文档维护原则

1. **保持更新**：每次迭代后及时更新相关文档
2. **记录决策**：重要的技术选型和业务决策要写清楚"为什么"
3. **实例优先**：多用代码示例和截图，少用抽象描述
4. **问题导向**：遇到的坑和解决方案要详细记录
5. **可复用性**：写文档时想着"下次能直接照着做"
