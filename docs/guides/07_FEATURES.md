## 1. 客户管理模块

### 1.1 功能概述

客户管理是系统的核心模块，用于管理从线索到签单的完整客户生命周期。

**核心功能**：
- 客户信息管理（新增、编辑、删除）
- 跟进记录管理
- 客户状态流转
- 客户分配与转移
- 客户搜索与筛选

### 1.2 客户状态流转

```
待跟进 → 已报价 → 已签单
   ↓        ↓
 已流失   已流失
```

**状态说明**：
- **待跟进**：新客户，需要持续跟进
- **已报价**：已经给客户发送报价单
- **已签单**：客户已签约，自动创建工地
- **已流失**：客户放弃或选择其他公司

**状态变更规则**：
- 任何状态都可以变更为"已流失"
- "已签单"后不能再变更为其他状态
- 状态变更会自动创建系统跟进记录
- 状态变更会发送通知给相关人员（小程序端）

### 1.3 跟进记录

**类型**：
- **手动跟进**：员工手动添加的跟进记录
- **系统跟进**：系统自动生成的记录（状态变更、分配变更等）

**显示规则**：
- 按时间倒序排列
- 支持换行显示
- 显示跟进人和跟进时间

**红点提示（小程序端）**：
- 有新跟进记录时显示红点
- 点开跟进 tab 后红点消失
- 通过 `lastFollowUpAt` 字段判断

### 1.4 客户编号规则

**显示优先级**：
- 优先显示 `customerNo`（业务编号）
- 如果没有 `customerNo`，显示 `_id`（数据库ID）
- 格式：`NO.{编号}`

**示例**：
```
NO.KH20260427001  （有业务编号）
NO.lead_001       （无业务编号，显示数据库ID）
```

### 1.5 使用场景

**场景1：新客户录入**
1. 销售接到客户咨询
2. 在系统中新建客户
3. 填写客户基本信息（姓名、电话、地址、预算等）
4. 分配负责人（默认为自己）
5. 添加初次跟进记录

**场景2：客户跟进**
1. 销售定期回访客户
2. 在客户详情页添加跟进记录
3. 记录沟通内容和客户反馈
4. 根据情况更新客户状态

**场景3：客户签单**
1. 客户决定签约
2. 将客户状态改为"已签单"
3. 系统自动创建工地
4. 发送全员通知

## 2. 工地管理模块

### 2.1 功能概述

工地管理用于管理装修项目的施工进度、工序验收、文件管理等。

**核心功能**：
- 工地信息管理
- 施工排期管理（8大节点）
- 工序验收
- 工地照片管理
- 工地文件管理
- 项目经理分配

### 2.2 施工排期（8大节点）

**节点列表**：
1. 拆除
2. 水电
3. 泥工
4. 木工
5. 油漆
6. 安装
7. 保洁
8. 验收

**排期规则**：
- 开工时自动生成排期
- 按工作日计算（排除周末和节假日）
- 每个节点有计划开始/结束时间
- 实际时间在验收时填写

**节点状态**：
- **未开始**：灰色，还未开始
- **施工中**：黄色，正在施工
- **已完成**：绿色，已验收完成

### 2.3 工序验收

**验收流程**：
1. 项目经理在小程序端提交验收
2. 上传验收照片
3. 填写验收备注
4. 提交后更新工序状态
5. 发送订阅消息通知相关人员

**验收内容**：
- 验收照片（必填）
- 验收备注（可选）
- 验收时间（自动记录）

### 2.4 工地分享功能

**功能说明**：
- 生成工地分享二维码
- 业主扫码可查看工地进度
- 非业主需要申请查看权限

**访问控制**：
- **业主**：手机号验证，自动通过
- **非业主**：需要提交申请，员工审批
- **员工**：直接访问，无需审批

**申请流程**：
1. 用户扫码进入分享页
2. 获取手机号
3. 验证是否为业主
4. 非业主提交申请
5. 发送通知给员工
6. 员工审批（通过/拒绝）
7. 发送审批结果通知给申请人

### 2.5 使用场景

**场景1：工地开工**
1. 客户签单后创建工地
2. 分配项目经理
3. 填写工地基本信息
4. 点击"开工"按钮
5. 系统自动生成施工排期
6. 发送开工通知

**场景2：工序验收**
1. 某个工序完成
2. 项目经理在小程序端提交验收
3. 上传现场照片
4. 填写验收备注
5. 提交后更新工序状态
6. 发送订阅消息

**场景3：业主查看进度**
1. 项目经理生成分享二维码
2. 发送给业主
3. 业主扫码进入
4. 授权获取手机号
5. 系统验证业主身份
6. 查看工地进度和照片

## 3. 报价管理模块

### 3.1 功能概述

报价管理用于创建、编辑、发送报价单。

**核心功能**：
- 报价单创建
- 报价项目管理
- 自动计算总价
- 报价单状态管理
- 附件上传

### 3.2 报价单状态

```
草稿 → 已发送 → 已接受
              ↓
            已拒绝
```

**状态说明**：
- **草稿**：正在编辑，未发送给客户
- **已发送**：已发送给客户，等待反馈
- **已接受**：客户接受报价
- **已拒绝**：客户拒绝报价

### 3.3 报价项目

**字段**：
- 项目名称
- 单位（项/平米/米等）
- 数量
- 单价
- 小计（自动计算）
- 备注

**计算规则**：
```
小计 = 数量 × 单价
总价 = 所有项目小计之和
```

### 3.4 使用场景

**场景1：创建报价单**
1. 在客户详情页点击"新建报价"
2. 添加报价项目
3. 填写单价和数量
4. 系统自动计算总价
5. 保存为草稿或直接发送

**场景2：修改报价单**
1. 打开报价单详情页
2. 点击编辑
3. 修改项目或价格
4. 保存更新

## 4. 材料与库存模块

### 4.1 功能概述

材料与库存模块分为两部分：
- **材料大厅**：管理产品目录（不涉及库存）
- **库存管理**：管理出入库流水和库存数量

### 4.2 材料大厅

**功能**：
- 新增材料
- 编辑材料信息
- 上架/下架材料
- 删除材料
- 材料搜索与筛选

**注意**：
- 材料大厅不显示库存数量
- 不能在这里做出入库操作
- 只是产品目录管理

### 4.3 库存管理

**功能**：
- 入库操作（增加库存）
- 出库操作（减少库存）
- 查看出入库流水
- 查看当前库存

**权限**：
- 仅 admin 和 manager 可以访问
- 其他角色看不到库存管理入口

**出入库规则**：
```
入库：currentStock = currentStock + quantity
出库：currentStock = currentStock - quantity
```

### 4.4 使用场景

**场景1：新材料入库**
1. 在材料大厅新建材料
2. 填写材料信息（名称、规格、单价等）
3. 保存后材料上架
4. 进入库存管理
5. 选择材料，点击"入库"
6. 填写入库数量和备注
7. 提交后更新库存

**场景2：工地领料**
1. 进入库存管理
2. 选择材料，点击"出库"
3. 选择关联工地
4. 填写出库数量
5. 提交后减少库存

## 5. 待办事项模块

### 5.1 功能概述

待办事项用于任务分配和提醒。

**核心功能**：
- 创建待办
- 分配执行人
- 设置截止时间
- 关联客户
- 标记完成/未完成
- 待办提醒

### 5.2 待办状态

- **pending**：待完成
- **completed**：已完成

### 5.3 关联规则

**可以关联**：
- 客户（线索）

**不能关联**：
- 工地（已移除此功能）

### 5.4 通知规则

**创建待办时**：
- 发送站内通知给执行人
- 发送订阅消息给执行人（小程序端）

**待办到期提醒**：
- 截止日期当天发送提醒（需要定时任务，暂未实现）

### 5.5 使用场景

**场景1：分配任务**
1. 管理员创建待办
2. 填写待办标题和内容
3. 选择执行人
4. 设置截止时间
5. 关联客户（可选）
6. 提交后发送通知

**场景2：完成任务**
1. 执行人在待办列表看到任务
2. 完成任务后标记为"已完成"
3. 记录完成时间

## 6. 通知中心模块

### 6.1 功能概述

通知中心用于接收系统通知和订阅消息。

**通知类型**：
- **站内通知**：在通知中心显示
- **订阅消息**：微信订阅消息推送（小程序端）

### 6.2 通知触发场景

| 场景 | 站内通知 | 订阅消息 |
|------|---------|---------|
| 新待办分配 | ✅ | ✅ |
| 客户状态变更 | ✅ | ✅ |
| 客户签单（全员） | ✅ | ✅ |
| 工地开工 | ✅ | ✅ |
| 工序验收 | ✅ | ✅ |
| 设计节点完成 | ✅ | ✅ |
| 查看权限申请 | ✅ | ✅ |
| 申请审批结果 | ✅ | ✅ |

### 6.3 订阅消息授权

**授权时机**：
- 登录时统一请求授权
- 关键操作前静默请求授权

**授权方式**：
```javascript
wx.requestSubscribeMessage({
  tmplIds: [模板ID列表],
  success: (res) => {
    // 用户授权
  }
})
```

### 6.4 使用场景

**场景1：查看通知**
1. 打开通知中心
2. 查看未读通知（高亮显示）
3. 点击通知标记为已读
4. 跳转到相关页面

**场景2：订阅消息**
1. 用户登录时授权订阅
2. 系统发生事件时触发通知
3. 调用云函数发送订阅消息
4. 用户收到微信推送

## 7. 组织架构模块

### 7.1 功能概述

组织架构用于管理员工账号和权限。

**核心功能**：
- 员工账号管理（新增、编辑、删除）
- 重置密码
- 停用/启用账号
- 角色分配

### 7.2 权限设计

**admin（管理员）**：
- 完整功能
- 可以新增/编辑/删除员工
- 可以重置密码
- 可以停用账号
- 显示所有列（账号、状态、入职时间等）

**非 admin（普通员工）**：
- 只读权限
- 只显示姓名、部门、电话
- 不显示账号、状态、操作按钮
- 不能进行任何管理操作

### 7.3 使用场景

**场景1：新员工入职（admin）**
1. 打开组织架构页面
2. 点击"新增员工"
3. 填写员工信息（账号、密码、姓名、角色等）
4. 保存后员工可以登录

**场景2：员工离职（admin）**
1. 找到员工记录
2. 点击"停用"按钮
3. 员工账号被停用，无法登录

**场景3：查看员工信息（非 admin）**
1. 打开组织架构页面
2. 只能查看姓名、部门、电话
3. 无法进行任何操作

## 10. 分享查看模块

### 10.1 功能概述

分享查看模块用于业主查看工地进度。

**核心功能**：
- 生成分享二维码
- 业主扫码查看
- 访问权限控制
- 申请审批

### 10.2 访问控制

**业主**：
- 手机号验证
- 自动通过

**非业主**：
- 需要申请
- 员工审批

**员工**：
- 直接访问
- 无需审批

### 10.3 申请流程

```
用户扫码 → 获取手机号 → 验证身份
                          ↓
                    是业主？
                    ↙    ↘
                  是      否
                  ↓       ↓
              直接查看   提交申请
                          ↓
                      员工审批
                      ↙    ↘
                  通过    拒绝
                  ↓       ↓
              可以查看  显示拒绝原因
```

### 10.4 使用场景

**场景1：业主查看进度**
1. 项目经理生成二维码
2. 发送给业主
3. 业主扫码
4. 授权手机号
5. 系统验证业主身份
6. 直接查看工地进度

**场景2：非业主申请查看**
1. 用户扫码
2. 授权手机号
3. 系统发现不是业主
4. 提示需要申请
5. 填写申请理由
6. 提交申请
7. 发送通知给员工
8. 员工审批
9. 发送审批结果
