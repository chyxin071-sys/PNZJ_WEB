# 测试与质量保障指南

> 本文档提供完整的测试清单、上线前检查、常见问题排查方法。

---

## 📋 测试流程总览

```
1. 功能测试 (每个功能开发完就测)
   ↓
2. 权限测试 (不同角色的访问控制)
   ↓
3. 边界测试 (异常情况、极端数据)
   ↓
4. 兼容性测试 (不同设备、浏览器)
   ↓
5. 性能测试 (加载速度、并发)
   ↓
6. 上线前检查 (配置、权限、数据)
```

## 第七部分：常见问题排查

### 7.1 登录问题

**问题**：无法登录
**排查步骤**：
1. 检查账号密码是否正确
2. 检查数据库 users 集合是否有该用户
3. 检查用户状态是否为 'active'
4. 检查网络请求是否成功
5. 检查 API 是否返回错误

**问题**：登录后自动退出
**排查步骤**：
1. 检查 token 是否过期
2. 检查 localStorage 是否被清空
3. 检查权限拦截逻辑

### 7.2 数据显示问题

**问题**：列表数据不显示
**排查步骤**：
1. 检查数据库是否有数据
2. 检查 API 请求是否成功
3. 检查数据格式是否正确
4. 检查前端渲染逻辑
5. 查看 Console 是否有错误

**问题**：数据显示不完整
**排查步骤**：
1. 检查是否超过 100 条限制
2. 检查分页逻辑
3. 检查字段是否存在

**问题**：时间显示错误
**排查步骤**：
1. 检查 createdAt 字段格式
2. 检查时间解析函数
3. 检查时区设置

### 7.3 排序问题

**问题**：列表排序混乱
**排查步骤**：
1. 检查排序字段（createdAt）格式
2. 检查排序逻辑（是否用字符串比较）
3. 使用统一的时间解析函数
4. 改用时间戳数值比较

**解决方案**：
```javascript
// 统一时间解析函数
function parseTime(time) {
  if (typeof time === 'number') return time
  if (time?.$date) return time.$date
  if (typeof time === 'string') return new Date(time).getTime()
  if (time instanceof Date) return time.getTime()
  return Date.now()
}

// 排序
data.sort((a, b) => parseTime(b.createdAt) - parseTime(a.createdAt))
```

### 7.6 权限问题

**问题**：非 admin 可以访问 admin 页面
**排查步骤**：
1. 检查 MainLayout 权限拦截逻辑
2. 检查 localStorage 中的 role 字段
3. 检查路由配置

**问题**：数据库操作失败（权限不足）
**排查步骤**：
1. 检查数据库集合权限设置
2. 检查用户是否登录
3. 检查云开发环境 ID 是否正确

### 7.7 性能问题

**问题**：页面加载很慢
**排查步骤**：
1. 检查数据量是否过大
2. 检查是否有不必要的请求
3. 检查图片是否过大
4. 使用浏览器开发者工具分析

**问题**：数据库查询很慢
**排查步骤**：
1. 检查查询条件是否合理
2. 检查是否需要建索引
3. 检查数据量是否过大
4. 考虑分页加载

```markdown
**优先级**：P0（严重）/ P1（重要）/ P2（一般）
```

### 9.2 Bug 修复步骤

1. **复现 Bug**：按照报告的步骤复现问题
2. **定位原因**：使用 systematic-debugging 排查
3. **修复代码**：修改相关代码
4. **测试验证**：确认 Bug 已修复
5. **回归测试**：确认没有引入新问题
6. **记录文档**：更新 CHANGELOG.md 和 ISSUES_AND_RISKS.md

**后端 Bug**
